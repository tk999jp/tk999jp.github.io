.NET 10 Desktop Runtime 導入ガイド

MidFD を起動するには、お使いの PC に .NET 10 Desktop Runtime (デスクトップ ランタイム) がインストールされている必要があります。

推奨手順: 最も簡単な導入方法

PCがインターネットに接続されている場合、アプリ起動時のエラーメッセージから直接導入するのが最も確実で簡単です。

  1. MidFD.exe を起動する
    ダウンロードして展開した MidFD.exe をダブルクリックして実行します。
  2. ダイアログの指示に従う
    「You must install .NET Desktop Runtime to run this application.」というダイアログが表示されたら、「Download it now」（はい）を選択します。
  3. インストーラーを実行する
    Webブラウザが起動し、自動的にインストーラーがダウンロードされます。
    ダウンロードされた windowsdesktop-runtime-10.0.x-win-x64.exe を実行します。
  4. インストール完了と再起動
    インストーラーの画面指示に従ってセットアップを完了させた後、再度 MidFD.exe を起動してください。

補助手段1: Microsoft公式ページから手動で導入する場合

手動でインストーラーを探して適用する手順です。

  1. Microsoftの公式ダウンロードページにアクセスします：
    .NET 10.0 ダウンロード (Microsoft公式)
  2. ページ内の 「.NET Desktop Runtime 10.0.x」 のセクションを探します。
    ※重要： 類似の「.NET SDK」や通常の「.NET Runtime」、「ASP.NET Core Runtime」と間違えないようにしてください。デスクトップアプリの動作には 「Desktop Runtime」 が必要です。
  3. OSのアーキテクチャに合わせたリンクをクリックしてダウンロードします。
    通常、64bit版の Windows をご利用の場合は 「x64」 を選択してください。
  4. ダウンロードされたインストーラーを実行し、画面の指示に従ってインストールを完了させます。

補助手段2: コマンドでのインストール (winget)

Windows 10 / 11 に標準搭載されているパッケージマネージャー「winget」を利用して、コマンドプロンプトや PowerShell からインストールすることも可能です。

PowerShell または コマンドプロンプトを開き、以下のコマンドを実行します：

winget install Microsoft.DotNet.DesktopRuntime.10

コマンド実行後、確認メッセージ等に従って処理が完了するのを待つだけで自動的にセットアップされます。

それでも起動しない場合の確認項目

ランタイムをインストールしたにもかかわらず起動しない場合は、以下の項目を確認してください。

トラブルシューティング
  • PCの再起動:
    ランタイムの反映がシステムに完全に適用されていない場合があります。一度PCを再起動してから再度 MidFD.exe を実行してみてください。
  • 別アーキテクチャ版の誤導入:
    x64版の Windows に対して Arm64版や x86(32bit)版のランタイムを導入していないか確認してください（通常は x64 を推奨します）。
  • セキュリティソフトによるブロック:
    セキュリティ対策ソフトや Windows SmartScreen が MidFD の実行を一時的にブロックしている場合があります。ファイルのブロック解除設定や例外設定を確認してください。
