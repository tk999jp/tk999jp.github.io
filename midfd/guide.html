Setup Roadmap

導入ロードマップ

ほとんどのユーザー環境では、以下のシンプルな3ステップだけで MidFD を使い始めることができます。

ほとんどの人はこの3ステップ

Step 1
通常版ZIPを取得

まずは配布元ページより、通常起動用のZIPファイルをダウンロードします。

通常版ZIPをダウンロード
※Source code (zip) は開発用ソース一式のため、通常の起動には使用できません。
Step 2
ZIPを展開

ダウンロードしたZIPファイルを右クリックしてすべて展開（解凍）します。迷ったら「ドキュメント」や任意の作業フォルダ等へ配置してください。

※書き込みエラーを避けるため、Program Files 等のシステム保護領域以外を推奨します。
Step 3
MidFD.exeを起動

展開フォルダ内の MidFD.exe を起動します。画面に従って好みの初期キー操作プリセットを選択すれば完了です。

※キー割り当ては後から設定画面でいつでも変更可能です。

うまく起動しない場合

万が一、上記の手順で起動できなかった場合は、以下の症状に合わせて対処を行ってください。

⚠️ Runtimeがない

「.NET Runtime が見つからない」という警告が出て起動しない状態。

【対処】: .NET 10 Desktop Runtime x64 をインストールします。
起動ダイアログの「はい」から自動ダウンロードするか、Microsoft公式から「x64版」を取得してください。
（または winget install Microsoft.DotNet.DesktopRuntime.10 を実行）
Microsoft ダウンロード
⚠️ 警告が出る

Windows SmartScreen やウイルス対策ソフトによって起動が一時ブロックされる状態。

【対処】: 個人開発プログラムのため、Windows保護機能が警告を出す場合があります。画面内の「詳細情報」をクリックし、入手元が公式であることを確認の上、ご自身の判断で「実行」を選択して起動してください。
⚠️ zipを間違えた

起動ファイル MidFD.exe が見当たらず、ソースコードのフォルダやファイルだけが入っている状態。

【対処】: 開発用の一式である Source code zip を取得した可能性があります。配布ページのダウンロードエリアに戻り、通常版の MidFD-win-x64.zip をダウンロードし直してください。

アーカイブ操作を便利にする 7-Zip、動画プレビューを可能にする ffmpeg、および外部テキストエディタ等の連携設定は、初回起動後にいつでも自由に行えます。

具体的なツール別の特徴や役割については、トップページの紹介をご覧ください。
